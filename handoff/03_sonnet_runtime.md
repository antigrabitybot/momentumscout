# [Sonnet仕様書] 推論ランタイム運用 v1.0

実行担当: Claude Sonnet 4.6（Vercel Serverless Function内から `claude-sonnet-4-6` としてAPI呼び出し）
役割: 「推論」ボタン押下時の原因分析エンジン（オンデマンド・都度課金）

## 役割の境界

- Sonnetは handoff/01 §2 のシステムプロンプトに**従う実行役**であり、
  プロンプト・スキーマ・確信度基準を自己判断で変えない
- web_searchは1推論あたり最大2回。検索クエリは「社名 急騰」「社名 材料 ニュース」
  を基本形とし、evidenceで裏が取れた場合は検索を省略してよい
- キーワード生成タスクは廃止された（機能Aは業種のみ）。Sonnetの仕事は本推論のみ

## 運用パラメータ（初期値）

| 項目 | 値 | 備考 |
|---|---|---|
| model | claude-sonnet-4-6 | 品質不足時のみFableへ格上げを検討 |
| max_tokens | 700 | スキーマ出力には十分 |
| web_search max_uses | 2 | コスト上限を兼ねる |
| 温度 | 既定 | 事実整理タスクのため調整不要 |

## 期待コストと監視

- 1回あたり: X検索 $0.10 + web_search $0.02 + トークン $0.02〜0.05 ≒ 15〜25円
- 月次でVercel KVの実行ログから実行回数・材料不明率・confidence分布を集計し、
  Fable監査（handoff/01 §5）へ提出する
- 材料不明率が10%未満に低下したら即Fableへエスカレーション（捏造傾向の兆候）

## 品質が出ない場合のエスカレーション順

1. evidenceパックの中身を確認（X検索0件が続く→クエリ形を調整）
2. プロンプトは触らずFableへ報告（プロンプト改訂はFableの専管）
3. それでも不足ならモデルをFableに切替えてコスト差を評価
